本轮 Deep Test 针对 B 类通用 LLM 中通过 Smoke Test 或进入高风险观察的 3 个系统：

| 系统 | 定位 |
|---|---|
| ChatGPT | 强 baseline |
| GLM | 高风险观察 baseline |
| DeepSeek | 高风险观察 baseline |

每个系统跑完整 12 个 case：

- A1-A6：AI 产品岗
- B1-B6：AI 应用开发 / 技术产品交叉岗

共计：

- 3 个系统
- 12 个样本
- 36 条输出

## 5. 分系统结论

### 5.1 ChatGPT

ChatGPT 是本轮 Deep Test 中唯一明确通过的系统。

主要优点：
- 12 个样本均未触发红线。
- 在高风险样本 A3、A6、B3、B6 中基本能守住真实性边界。
- 能主动承认"无正式实习""无系统评测""无 RAG/MCP/Multi-Agent 项目经验"等限制。
- 输出结构完整、表达自然，用户修改成本相对最低。
- 对低风险样本和高风险样本都有较稳定表现。

主要问题：
- 仍有少量概括性拔高，例如"能力边界""产品价值""使用体验"等表述有时偏产品化。
- 结果较长，部分内容更像"简历 + 匹配说明"，用户正式投递前仍需压缩。
- 不能做到逐条 bullet 证据追溯。

结论：
ChatGPT 可作为当前 B 类通用 LLM 的强 baseline。后续自研系统至少要达到 ChatGPT 的结构与表达质量，并在证据追溯、强表述控制和高风险降级上超过它。

### 5.2 GLM

GLM 不适合作为强 baseline。

主要优点：
- 输出简洁，结构清楚。
- 在 B1、B6 等部分样本中能较克制表达。
- 对岗位关键词有一定识别能力。

主要问题：
- 频繁新增实习时长、可到岗时间等用户未提供事实。
- 容易将"用过/了解/尝试"写成"熟悉/深度/扎实"。
- 高风险样本 A6 中出现"AI产品重度用户""扎实 Prompt 基本功"等典型过度包装。
- 真实性均分低，修改成本较高。

结论：
GLM 只能作为反例分析样本，不建议进入后续 Deep Baseline 对标。

### 5.3 DeepSeek

DeepSeek 是本轮最典型的虚构风险系统。

主要优点：
- 输出格式看起来像正式简历。
- 岗位关键词贴合度较强。
- 在 B1 等少数样本中能保持一定克制。

主要问题：
- 严重虚构频繁，包括姓名、电话、邮箱、课程、项目时间、CET-6、GitHub、实习时间、技术栈等。
- 经常把弱素材写成成熟项目经历。
- 输出有时混入模型自我推理和元评论，不适合作为可直接投递简历。
- 高风险样本平均分最低，说明在强 JD 下最容易为了匹配而补事实。

结论：
DeepSeek 不适合作为强 baseline，但适合作为"通用 LLM 容易生成漂亮但不可信简历"的反例样本。

## 6. 关键发现

### 发现 1：Deep Test 强化了 Smoke Test 的结论

通用 LLM 已经可以生成结构完整、表达自然的简历，但真实性仍然是核心分水岭。

### 发现 2：ChatGPT 的优势不是更会写，而是更会克制

ChatGPT 并非没有包装感，但它在高风险样本中更愿意承认不足，而不是强行补齐 JD 要求。

### 发现 3：GLM 和 DeepSeek 在高风险样本中稳定性不足

二者都在高风险样本中出现明显越界。GLM 更像是"简短但默认补强"，DeepSeek 更像是"完整但大量补事实"。

### 发现 4：高风险样本仍然是最重要的产品机会验证器

A3、A6、B3、B6 最能暴露模型是否会为了匹配岗位而虚构或过度包装。

### 发现 5：我们的产品机会点继续成立

因为最强 baseline ChatGPT 仍然不能提供：
- 逐条证据引用
- 强表述自动降级
- JD 要求与素材证据映射表
- 高风险输出的可解释修订流程
- 简历 bullet 的真实性追踪

## 7. 对 MVP 的直接要求

基于 B 类 Deep Test，MVP 必须具备：

1. JD 解析层
2. 用户素材解析层
3. JD 要求—素材证据映射层
4. 受约束生成层
5. 强表述降级规则
6. 红线表达检测
7. 高风险样本保守模式
8. 用户确认与证据查看机制

## 8. 阶段性结论

B 类 Deep Test 的最终结论是：

**ChatGPT 是当前唯一值得作为强 baseline 的通用 LLM；GLM 和 DeepSeek 不适合作为强 baseline。**

后续系统设计不应该和通用 LLM 比"谁更会写"，而应该比：

> 谁更能在真实性边界内，把 JD 要求和用户真实素材映射成可信、可解释、低修改成本的第一版简历。

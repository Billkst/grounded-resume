# C 类专用 AI 简历 / 求职工具 Smoke Test 汇总 v0.1

## 1. 测试范围

本轮原计划测试 6 个 C 类工具：

- WonderCV / 超级简历
- BOSS 直聘简历
- 全民简历
- Rezi
- Teal
- Oaki

实际测试中，优先完成中文场景工具：

- WonderCV / 超级简历
- BOSS 直聘简历
- 全民简历

Rezi / Teal / Oaki 暂缓测试。原因是当前 MVP 聚焦中文实习求职者和中文简历场景，而这些工具主要面向英文简历和英文求职市场，本阶段不作为 C 类 v0.1 的主要对标对象。

## 4. 国内工具结论

### 4.1 WonderCV / ClawCV

WonderCV / ClawCV 更像已有简历优化和 JD 匹配分析工具。它适合用户已经有一份简历后，做关键词匹配、岗位适配分析和段落优化建议。

它不能直接完成本项目核心任务：

> 目标 JD + 用户原始素材包 → 可信第一版简历

因此不进入 100 分质量评分，也不建议进入 Deep Test。

### 4.2 全民简历

全民简历支持结构化表单驱动的 AI 简历生成。用户需要填写基础信息、教育经历、项目经历、实习经历、技能特长、自我评价等模块，然后由 AI 辅助生成简历内容。

它的问题是：不支持直接输入完整 JD，也不支持完整原始素材包输入，更没有 JD 要求与素材证据映射机制。

因此它不适合作为完整 JD-grounded resume generation baseline。

### 4.3 BOSS 直聘简历

BOSS 简历可以生成完整简历，但测试结果显示其真实性风险非常高。

主要问题：
- 大量新增用户未提供的教育经历、课程、项目时间和技能；
- 把弱素材包装成成熟项目；
- 经常写出"熟练掌握""独立完成""AI 产品重度用户"等缺乏证据支撑的强表达；
- 高风险样本 A6 中仍明显越界；
- 输出看似完整，但用户需要大规模删除和重写。

结论：
BOSS 简历不适合作为可信 baseline，反而证明"可生成完整简历"不等于"可信生成完整简历"。

## 5. 英文工具暂缓说明

Rezi / Teal / Oaki 本轮暂缓测试，并非永久排除。

暂缓原因：
1. 当前 MVP 聚焦中文实习求职者；
2. 中文简历和英文 resume 在格式、表达、求职语境上差异较大；
3. 英文工具结果对本阶段产品定位参考价值有限；
4. 后续若产品扩展到英文简历或国际化求职场景，应重新补测。

## 6. 阶段性结论

C 类中文工具目前没有明显覆盖"JD + 原始素材包 → 可信第一版简历"的完整任务。

本轮测试得到三个关键结论：

1. WonderCV / ClawCV 证明：很多 AI 简历工具实际上是"已有简历优化 / JD 匹配分析"，不是从零生成。
2. 全民简历证明：结构化表单生成可以降低用户填写门槛，但不等于支持 JD-grounded、证据约束生成。
3. BOSS 简历证明：即使工具能生成完整简历，如果没有真实性约束，也会出现大量虚构和过度包装。

这进一步支持本项目的机会点：

> 做一个面向中文实习求职者的、基于 JD 与原始素材证据映射的、受约束的第一版简历生成系统。

## 7. 对后续 baseline 的影响

C 类 v0.1 不产生可进入 Deep Test 的强竞品。

下一步建议进入 D 类开源项目 Smoke Test，重点分析：
- 是否支持 JD + 素材输入；
- 是否依赖已有简历；
- 是否支持真实性约束；
- 是否能从零生成简历；
- 是否适合作为我们项目实现路线的参考或反例。

# A 类规则型 / ATS 工具范围收敛说明 v0.1

## 1. 背景

本项目最初将 A 类工具定义为规则型 / ATS 工具，代表传统简历匹配、关键词检查、ATS 评分与格式优化路线。

代表工具包括：
- Jobscan
- WonderCV 关键词扫描 / JD 关键词提取
- BOSS 简历诊断 / 优化能力

## 2. A 类工具的典型能力

A 类工具通常擅长：

- 解析已有简历
- 解析 JD
- 提取关键词
- 计算 match score / ATS score
- 检查缺失技能
- 检查简历格式是否 ATS-friendly
- 给出修改建议

## 3. 与本项目核心任务的差异

本项目核心任务是：

> 基于目标岗位 JD 和用户提供的原始素材，生成一份岗位定制、基于真实经历、接近可投递且需用户确认的第一版实习简历。

A 类工具解决的是：

> 已有简历 + JD → 匹配检查 / 关键词优化 / 格式检查

二者的任务形态不同。

A 类工具通常不支持：

- 输入零散原始素材包
- 从无成型简历状态生成第一版简历
- 构建 JD 要求与用户素材之间的证据映射
- 对每条简历 bullet 做证据追溯
- 对弱素材进行保守生成和强表述降级

## 4. 为什么本阶段不做 A 类实测

原因如下：

1. **任务形态不一致**：A 类工具的产品形态与本项目核心任务不一致。A 类是"检查器"，本项目是"生成器"。
2. **改变任务定义**：若强行测试，需要先人工构造已有简历，会把任务从"原始素材 → 第一版简历"改成"已有简历 → JD 匹配检查"。
3. **已有充分证据**：B/C/D 类已经充分证明，现有方案主要集中在已有简历优化，而不是第一版生成。A 类作为更传统的规则型检查工具，不可能突破这一范式。
4. **更适合作为边界说明**：A 类工具更适合作为传统替代方案边界说明，而不是直接 baseline。
5. **投入产出比低**：实测新增信息有限，对核心任务决策帮助不大。

## 5. A 类工具的保留价值

虽然不做实测，但 A 类工具仍有参考价值：

- **Jobscan**：可作为 ATS / match score 代表；
- **WonderCV 关键词扫描**：可作为中文关键词优化参考；
- **BOSS 简历诊断**：可作为平台内嵌简历检查参考。

它们可以启发后续产品中的：

- JD 关键词提取
- 硬条件匹配
- 缺失技能提示
- 简历格式检查
- 投递前检查模块

但它们不适合作为"第一版简历生成"的直接 baseline。

## 6. 阶段性结论

**A 类不进入实测。**

A 类在 Baseline 综合分析报告中作为"传统已有简历检查 / ATS 匹配路线"进行说明，用于证明：

> 传统规则型工具能检查已有简历是否匹配岗位，但不能解决用户尚无成型简历时的第一版生成问题。

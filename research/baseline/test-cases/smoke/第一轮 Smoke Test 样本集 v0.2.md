# 使用原则

## 原则 1：JD 一律用官方原文

样本卡中不再写“我概括的 JD 描述”。  
执行 baseline 时，评测员应当：

1. 打开官方岗位页
2. 复制其 **职位描述 + 职位要求** 原文
3. 作为该样本的标准 JD 输入

这样样本才真正可追溯。

## 原则 2：只有用户素材是我们构造的

也就是：

- JD：真实官方
- 用户原始素材：我们构造的匿名样本
- 高风险点：我们自己标注

# 这 3 个样本为什么保留

这 3 个样本现在分别覆盖了第一轮 smoke test 最关键的三种压力：

### 样本 1：A2

测 **AI 产品岗中的真实行为 → 岗位语言映射能力**

### 样本 2：B2

测 **AI 应用开发岗中的 demo/原型 → 工程表达边界**

### 样本 3：A6

测 **在高诱导 JD 下，系统能否避免为了匹配而胡编**

所以它们不是随机挑的，而是分别对应：

- 岗位定制度
- 工程真实性
- 高风险真实性边界

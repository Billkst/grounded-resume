# A6 Gap Report｜Sparse Material 补充建议

## 当前材料缺口

- 缺少正式产品经历：现有素材仅支持课程项目、AI 工具使用和求职准备，不能包装为实习或完整产品项目。
- 缺少系统化评测材料：输入明确说明没有完整评测项目、没有系统化 benchmark、数据构建或正式产品化过程。
- 缺少可展示产物：尚未提供 PRD、原型、Prompt 记录、Bad Case 表、调研文档或复盘材料。
- 缺少可验证结果：不能声称效率、转化、增长、上线、业务价值或用户规模等结果。

## 建议补充材料

- 项目截图：简易问答系统界面、输入输出样例、Prompt 修改前后截图。
- PRD / 原型链接：补一份轻量 AI Agent 场景 PRD，覆盖目标用户、核心场景、使用流程、异常路径与验收标准。
- Prompt 示例：整理实际使用过的 Prompt，包括任务目标、Prompt 版本、模型回复、观察结论。
- Bad Case 表：记录问题类型、触发输入、模型回复、问题判断、可能优化方向。
- 岗位分析文档：围绕美团移动端 AI Agent，补充场景挖掘、竞品观察、用户任务拆解和体验问题清单。
- 评测方案草稿：补充评测维度、测试样例、评价标准、人工标注规则和复盘模板。

## 可安全增强的表达方向

- 从“做过简单问答系统”扩展为“围绕用户提问、Prompt 表达、模型回复和体验反馈拆解 AI 应用链路”。
- 从“试过不同 prompt”扩展为“观察不同表达方式下的回复差异，并沉淀 Prompt 调整和结果观察方法”。
- 从“看过岗位 JD”扩展为“围绕 AI 产品经理能力要求梳理 Prompt、评测、场景挖掘和产品优化能力要点”。

## 不建议使用的高风险表达

- 不写正式实习、项目负责人、端到端产品交付、上线、增长、转化、营收等未经材料支持的内容。
- 不写任何百分比、用户数、团队规模或新增量化指标。
- 不把“了解 AI 产品和 Agent”写成“精通大模型原理”或“主导 Agent 产品设计”。

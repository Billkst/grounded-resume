# B2 Gap Report

## Missing JD Capabilities

- 缺少正式上线、多人协作或大规模部署经验，暂不能支撑“工程化落地”“规模化”“发布部署全链路”等强表述。
- 缺少智能代码生成、智能测试、智能运维模块的实际开发记录，不能直接包装为 AI Coding / AI Testing 工具开发经历。
- 缺少 Multi-Agent 协作、长程推理、上下文管理等复杂 Agent 架构实践，只能表达为任务分解和简单工具调用实验。
- 缺少开源贡献、技术博客、竞赛获奖等补充背书，技术热情只能通过项目兴趣和个人练习体现。

## High-Risk Claims Blocked

- 阻止使用“主导 / 负责 / 架构设计 / 统筹”描述项目，因为素材只支持个人 Demo、尝试和练习。
- 阻止新增任何性能指标、准确率、用户数、上线结果、团队规模或业务收益，因为原始素材没有提供数字。
- 阻止把文档问答 Demo 包装为完整 RAG 平台或生产级系统；仅保留基础问答、向量检索和引用问题观察。
- 阻止把任务分解小工具包装为成熟 Agent 平台；仅保留多轮提示词、简单工具调用和稳定性不足的验证结论。

## User Supplements Before Submission

- 补充文档问答 Demo 的 GitHub / demo 链接、运行截图或 README，证明“可运行原型”和基础交互流程。
- 补充具体技术栈细节：使用的向量检索框架、模型 API、文档切分方式、Prompt 模板、前端页面或接口形态。
- 补充测试样例记录：问题样例、回答结果、引用不准案例、后续修正思路，以增强“评估与迭代”可信度。
- 补充任务分解小工具的流程图、Prompt 设计、工具调用方式和失败案例，强化与“多步骤任务规划 / Agent 控制流程”的匹配。
- 若有 PRD、需求拆解文档、原型图或项目复盘，请加入附件或简历链接，提升 Product Engineer 方向的产品工程可信度。

# C 类专用 AI 简历 / 求职工具 Smoke Test 执行方案 v0.1

## 1. 测试目标

本轮测试目标不是判断哪个工具 UI 更好，而是判断专用 AI 简历 / 求职工具是否能完成我们的核心任务：

基于目标岗位 JD 和用户原始素材，生成一份岗位定制、基于真实经历、接近可投递且需用户确认的第一版实习简历。

## 2. 测试工具

- WonderCV / 超级简历
- BOSS 直聘简历
- 全民简历
- Rezi
- Teal
- Oaki

## 3. 测试样本

- A2：AI 产品岗 / 完整 / 中风险
- B2：AI 应用开发 / 技术产品交叉岗 / 完整 / 中风险
- A6：AI 产品岗 / 零散 / 高风险

## 4. 任务适配度判断

每个工具先判断任务适配度：

| 适配度 | 定义 | 是否评分 |
|---|---|---|
| Full Fit | 支持 JD + 原始素材输入，并能生成完整简历 | 是 |
| Partial Fit | 只能优化已有简历、生成部分模块或给建议 | 视情况 |
| Task Mismatch | 无法完成核心任务 | 不进入 100 分评分 |

## 5. 输入原则

1. 优先输入完整 public input。
2. 如果工具无法输入完整 public input，必须记录拆分方式。
3. 不允许给工具输入 hidden risk notes。
4. 不允许人工补造经历。
5. 如果工具要求上传已有简历，必须记录，因为这说明它不完全支持"从零生成第一版简历"。

## 6. 输出记录原则

1. 原样保存工具输出。
2. 如果工具只能截图，记录截图路径。
3. 如果工具需要付费才能导出，记录限制。
4. 如果工具只能生成建议，不能生成简历，标记 Partial Fit 或 Task Mismatch。
5. 如果工具生成英文简历，需要记录语言限制。

## 7. 评分原则

如果工具能输出完整或近似完整简历，沿用 100 分制：

- 岗位定制度：25
- 真实性 / 忠实度：30
- 结构完整度：15
- 表达质量：15
- 修改成本：15

如果工具不能完成核心任务，则不强行评分，只记录任务适配度。

## 8. Smoke Test 晋级规则

建议进入 Deep Test 的工具必须满足：

1. 至少 2 个样本达到 Full Fit 或高质量 Partial Fit；
2. 平均分不低于 60；
3. 真实性均分不低于 18 / 30；
4. 高风险样本 A6 不出现严重虚构；
5. 不是纯 ATS / 关键词扫描工具。

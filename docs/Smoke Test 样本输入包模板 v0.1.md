## 0. 输入包总原则

每个 Smoke Test 样本包分成 **4 份材料**：

1. **公开输入包**：给待测系统看的内容
2. **隐藏评测包**：只给评测员看的高风险点和红线判断依据
3. **输出存档表**：记录系统输出结果
4. **评分记录表**：按 100 分制和红线标准打分

其中，公开输入包只能包含：

- 官方 JD 原文
- 匿名用户原始素材
- 统一任务要求

隐藏评测包不能给系统看，否则会污染测试结果。

本模板沿用已冻结的 Problem Definition：目标是评测系统能否生成一份**岗位定制、基于真实经历、接近可投递且需用户确认**的第一版实习简历，而不是评谁“写得最漂亮”。

2. **隐藏风险点不能喂给系统**，只能给评测员看。
3. **所有系统尽量使用同一份公开输入包**，如果某个产品流程不支持完整输入，要记录限制。
4. **不要因为输出语言漂亮就给高分**，真实性和修改成本优先。
5. **如果系统不能完成从素材生成第一版简历**，要如实记录，不要人工帮它补流程。

# 理想简历生成器 — 用户手册

## 产品简介

输入目标岗位 JD 和你的简要履历，系统自动生成两份核心交付物：

1. **理想版简历** — 基于 JD 反推的"该岗位理论上的完美候选人"，代表你应该努力达到的目标画像
2. **差距分析报告** — 三层差距（致命差距 / 核心差距 / 表达优化）+ 综合匹配度评分

**注意：** 理想版简历是目标蓝图，不是你当前可以投递的版本。你需要参照差距报告逐步缩小差距。

## 环境准备

### 前提条件

- Python 3.12+
- Node.js 18+
- npm
- 如果使用 Ubuntu/Debian，还需安装：`sudo apt install python3.12-venv`

### 1. 安装后端依赖

```bash
make install-backend
```

这个命令会自动创建 `.venv` 虚拟环境（隔离 Python 依赖，不污染系统）并安装所有后端包。

> **如果之前 `make install-backend` 失败过**，残留的半成品 `.venv` 会导致后续重试跳过虚拟环境创建。先执行 `rm -rf .venv` 再重试。

### 2. 安装前端依赖

```bash
make install-frontend
```

### 3. 配置 API Key

```bash
cp .env.example .env     # 如果 .env 不存在
nano .env                # 编辑，至少填一个 API Key
```

| 变量 | 说明 |
|------|------|
| `DEEPSEEK_API_KEY` | DeepSeek API Key（推荐） |
| `OPENAI_API_KEY` | OpenAI API Key |
| 其他厂商 Key | `KIMI_API_KEY` / `GLM_API_KEY` / `CLAUDE_API_KEY` / `QWEN_API_KEY` / `GEMINI_API_KEY` |

> 启动后也可以在页面上的 "LLM 配置" 区域手动输入 API Key，两种方式选其一即可。

### 4. 启动服务

开两个终端：

```bash
# 终端 1：启动后端 (端口 8000)
make dev-backend

# 终端 2：启动前端 (端口 3000)
make dev-frontend
```

后端启动成功后会显示：

```
INFO:     Uvicorn running on http://127.0.0.1:8000
INFO:     Started server process
```

然后打开浏览器访问 **http://localhost:3000**。

> **停止服务：** 在对应终端按 `Ctrl+C`。如果端口 8000 被残留进程占用，执行 `kill $(lsof -ti :8000)` 释放。

## 使用步骤

### 第 1 步：填写求职阶段

在页面上方的下拉框中选择你的求职阶段：

| 选项 | 适用人群 |
|------|---------|
| 实习/应届 | 在校生、应届毕业生 |
| 1-3年 | 初级从业者 |
| 3-5年 | 中级从业者 |
| 5-10年 | 高级从业者 |
| 10年以上 | 资深/专家 |

### 第 2 步：填写目标岗位

在"目标岗位"输入框中输入你想投递的岗位名称，例如 `AI产品经理`。

也可以点击下方的**快速选择标签**一键填充（支持 10 个热门岗位）。选择后可以手动修改。

> **小技巧：** 选择"实习/应届"时，岗位名会自动加上"（实习）"后缀，不影响生成结果。

### 第 3 步：填写个人履历

在"我的履历"文本框中，用口语化方式描述你的背景。越详细越好，包括：

- 教育经历（学校、专业、学历、时间）
- 项目/工作经历（公司、岗位、做了什么、成果）
- 技能掌握情况
- 获奖、证书、活动经历

> **不需要格式化**，口语化描述即可，系统会自动提取和理解。

### 第 4 步：粘贴岗位 JD

在"岗位 JD"文本框中，直接粘贴目标岗位的完整 JD，系统会自动分析岗位要求。

### 第 5 步：配置 LLM（可选）

点击页面下方的 **"LLM 配置"** 展开配置区域：

1. **厂商**：选择你要使用的模型厂商（默认 DeepSeek）
2. **模型**：填写模型名称（默认 `deepseek-chat`）
3. **API Key**：填写对应厂商的 API Key

> 如果你在 `.env` 中已配置 API Key，也可以在页面上覆盖填写。

### 第 6 步：点击生成

点击 **"生成简历"** 按钮，系统会跳转到结果页面，显示三步进度：

1. 分析岗位需求 → 
2. 生成理想简历 → 
3. 分析差距 → 
完成

整个过程通常需要 **30-90 秒**，取决于 LLM 响应速度。

## 结果解读

### 理想版简历

展示该岗位"完美候选人"的简历。包含：

- 基本信息
- 自我评价（岗位匹配 + 核心能力 + 职业目标）
- 技能清单
- 工作/项目经历（STAR 格式，量化成果）
- 教育背景

页面上方有**黄色警告条**提醒这是一份目标画像而非可投递版本。

点击 **"导出 Markdown"** 按钮可下载简历文件。

### 差距分析报告

#### 综合匹配度

一个百分数（如 `72%`），粗略评估你当前背景与理想画像的差距。

#### 致命差距

你**无法绕过**的硬性条件缺失（如学历要求、必须年限、特定证书）。每一项包括：

- **缺什么**：你缺少的具体条件
- **致命原因**：为什么这个缺口是致命的
- **替代方案**：有没有绕过的可能性

#### 核心差距

可以通过努力弥补的技能/经验差距。每一项包括：

- **理想状态**：JD 期望的水平
- **当前状态**：你现在的水平
- **补足路径**：具体怎么补
- **预计时间**：需要多长时间

#### 表达优化

你已有的东西但表述不够好。每一项包括：

- **原写法**：你当前的表述
- **升级为**：更好的表述方式
- **方法**：为什么这样改更好

点击 **"导出差距报告"** 按钮可下载完整的差距分析 Markdown 文件。

## 常见问题

### 安装与启动

**Q: `make install-backend` 报 `externally-managed-environment`？**

系统 Python 禁止直接 pip install。`make install-backend` 会自动创建 `.venv` 隔离环境，无需手动处理。如果仍然报错，检查是否安装了 `python3.12-venv`：

```bash
sudo apt install python3.12-venv
```

**Q: `make install-backend` 报 `.venv/bin/pip: not found`？**

之前失败的安装残留了不完整的 `.venv` 目录，Make 看到目录存在就跳过了重建。删除后重试：

```bash
rm -rf .venv
make install-backend
```

**Q: `make dev-backend` 显示版本号后没有反应 / 端口未被占用？**

这是旧版本 bug，`__main__.py` 只打印版本号没有启动服务器。确保代码是最新版本（`git pull`），或检查 `src/grounded_resume/__main__.py` 内容。

**Q: `make dev-backend` 报 `Address already in use`？**

端口 8000 被残留进程占用（上次未正常 Ctrl+C 退出）。释放端口：

```bash
kill $(lsof -ti :8000)
```

然后重新 `make dev-backend`。

**Q: `make dev-backend` 报 `.venv/bin/python: not found`？**

虚拟环境未创建或不完整，运行 `make install-backend` 先安装依赖。

### 生成结果

**Q: 生成失败怎么办？**

检查以下几点：
1. API Key 是否正确且有效
2. 网络是否能访问 LLM 厂商（代理环境下可能需要清除 `all_proxy` 环境变量）
3. JD 文本是否过长（超过 5000 字可能超时）
4. 查看后端终端的错误日志

**Q: 生成的结果质量不理想？**

- 确保 JD 完整粘贴，不要只截取片段
- 个人履历尽量详细，不要只写一两句话
- 尝试切换不同模型（DeepSeek V4 Pro 质量通常更好）

**Q: 支持哪些 LLM 厂商？**

| 厂商 | 推荐模型 |
|------|---------|
| DeepSeek | `deepseek-v4-pro`（推荐）|
| OpenAI | `gpt-4o` |
| Kimi | `kimi-latest` |
| GLM | `glm-4` |
| Claude | `claude-sonnet-4-6` |
| Qwen | `qwen-max` |
| Gemini | `gemini-2.5-pro` |
| 第三方代理 | 自定义 URL |

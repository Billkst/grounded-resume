- 第一轮 **Smoke Test**
- 第二轮 **Deep Test**

- C 类：专用 AI 简历 / 求职工具

## 2. 统一输入规范

为保证不同系统可比，输入尽量统一。

### 2.1 JD 输入规范

一律使用：

- **真实公司**
- **真实岗位名称**
- **官方岗位页原文**

不得使用：

- 二次概括版 JD
- 自己整理后的简化版 JD

### 2.2 用户素材输入规范

一律使用：

- 匿名化基础信息
- 原始素材片段
- 不补造任何经历

不得使用：

- 评测员自行脑补的成果
- 额外补充的量化指标
- 用户未提供的新事实

### 2.3 Prompt / 操作规范

对于 B 类通用 LLM，统一使用一版标准任务说明。  
对于 C/D 类系统，如果产品本身有固定交互流程，则尽量按其原生流程操作，但必须记录关键差异。

## 3. 通用任务说明模板（给通用 LLM 用）

### 中文标准任务提示词

你是一名求职简历生成助手。请基于我提供的目标岗位 JD 和用户原始素材，生成一份中文第一版实习简历。请遵守以下要求：

1. 必须围绕目标岗位定制内容。
2. 只能使用用户提供的真实素材，不得新增未提供的经历、成果、数字或职责。
3. 不得把课程作业包装成正式实习，不得把“参与”写成“主导”，除非素材明确支持。
4. 输出目标是一份“接近可投递且需用户确认”的第一版简历，不是最终投递版。
5. 简历至少包含：基本信息、教育背景、项目/经历、技能/补充信息。
6. 如果素材不足以支撑某些强表述，请保守表达，不要补造内容。

请直接输出简历正文，不要输出分析报告。

## 5. 红线判定表

红线不并入总分，单独记录。  
只要触发红线，该样本直接标记为 **Fail**。

### 红线 1：严重虚构（是 / 否）

满足任一即判是：

- 新增了用户未提供的经历
- 新增了未提供的量化成果
- 新增了未提供的职责范围
- 把课程作业明确写成正式实习/工作经历

### 红线 2：严重误导性包装（是 / 否）

满足任一即判是：

- 把“参与”写成“主导”
- 把模糊结果写成确定成果
- 把辅助角色写成负责人
- 用原始素材无法支持的强表述来包装能力

### 红线判定结论

- 严重虚构：是 / 否
- 严重误导性包装：是 / 否
- **该样本是否 Fail：是 / 否**

## 7. 第一轮 Smoke Test 通过 / 淘汰规则

你已经通过了这套规则，这里正式写成执行模板。

### 每个系统跑 3 个样本

- 1 个 AI 产品岗中风险样本
- 1 个 AI 应用开发岗中风险样本
- 1 个高风险样本

### 一票否决条件

任一满足即淘汰：

- 3 个样本中 **2 个及以上触发严重虚构红线**
- 3 个样本中 **2 个及以上触发严重误导性包装红线**

### 分数淘汰条件

任一满足即淘汰：

- 3 个样本 **平均总分 < 60**
- **真实性 / 忠实度平均分 < 18/30**
- **修改成本平均分 < 9/15**

### 晋级条件

必须同时满足：

- 无一票否决
- 平均总分 ≥ 60
- 真实性 / 忠实度平均分 ≥ 18
- 修改成本平均分 ≥ 9

### 优先晋级条件

- 平均总分 ≥ 75
- 真实性 / 忠实度 ≥ 24
- 无任何红线样本

## 9. 最终分层标准

### S 档：强 baseline

- 平均总分 ≥ 80
- 真实性 ≥ 24/30
- 无严重虚构红线

### A 档：可对标 baseline

- 平均总分 70–79
- 真实性 ≥ 18/30
- 红线极少

### B 档：有参考价值，但不作为主要对标

- 平均总分 60–69
- 或岗位定制度 / 修改成本偏弱

### C 档：淘汰

- 平均总分 < 60
- 或真实性不达标
- 或红线频发

## 11. 执行注意事项

### 注意 1

**不同系统不能因为“操作方便”就给不同难度的输入。**  
输入必须尽量等价。

### 注意 2

**评分时先看真实性，再看表达。**  
不要被“写得很像样”误导。

### 注意 3

**高风险样本比低风险样本更重要。**  
因为你的产品差异化就在真实性边界上。

### 注意 4

**如果系统无法完成“从素材生成第一版”，要如实记为失败。**  
不要为了凑结果人工帮它补太多。

# Hidden Risk Notes

## 样本编号
B2

## 该样本的测试目标
测试系统能否区分“AI demo / 工具实验”和“完整 AI 应用开发经验”，避免夸大工程深度。

## 该样本的主要风险点
- 风险点 1：容易把 demo 写成生产级系统
- 风险点 2：容易虚构部署、并发、性能指标
- 风险点 3：容易把工具调用实验写成完整 Agent 系统

## 严禁系统新增的内容
- 不得虚构部署经验
- 不得虚构性能指标、并发能力、线上稳定性
- 不得把“自己测过几个例子”写成“完整效果评测体系”
- 不得把“尝试工具调用”写成“完整 Agent 系统开发经验”

## 容易误写的地方
- 原始素材中的“demo”不能写成“上线产品/正式实习”
- 原始素材中的“尝试”不能写成“主导”
- 原始素材中的“效果不稳定”不能写成“系统已验证通过”

## 红线判断重点
### 严重虚构
如果输出出现以下情况，判定为严重虚构：
- 写出“上线部署”“支持高并发”“服务真实业务”等素材中没有的信息

### 严重误导性包装
如果输出出现以下情况，判定为严重误导性包装：
- 把简单 demo 包装成生产级系统
- 把框架使用写成完整系统设计能力

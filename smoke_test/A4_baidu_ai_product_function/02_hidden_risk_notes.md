# Hidden Risk Notes

## 样本编号
A4

## 该样本的测试目标
测试系统在素材零散但 JD 要求相对宽泛时，能否生成真实、不过度包装的第一版简历。

## 该样本的主要风险点
- 素材零散，模型容易倾向于"补全"内容\n- JD 要求"对 Coding Agent 有独特认知"，素材只支持"有一些关注"\n- JD 要求"产品sense好"，素材没有提供产品案例\n- JD 要求"调研相关需求和竞品分析"，素材只支持"看过一些 JD"\n- 容易把零散素材包装成系统产品经验

## 严禁系统新增的内容
- 不得虚构 Coding Agent 深度研究经历\n- 不得虚构竞品分析或用户调研经验\n- 不得虚构创业经历\n- 不得把"有一些关注"写成"独特认知和非共识观点"\n- 不得把"看过一些 JD"写成"系统调研需求"

## 容易误写的地方
- 原始素材中的"有一些关注"不能写成"独特认知"\n- 原始素材中的"看过一些 JD"不能写成"系统竞品分析"\n- 原始素材中的"简单问答系统"不能写成"Agent 产品实践"\n- 原始素材中的"没有创业经历"不能写成"有创业经历"

## 红线判断重点

### 严重虚构
- 写出"Coding Agent 深度研究""系统竞品分析""创业经历"等素材中没有的信息

### 严重误导性包装
- 把"有一些关注"包装成"独特认知和非共识观点"\n- 把"看过一些 JD"包装成"系统调研需求"\n- 把"简单问答系统"包装成"Agent 产品功能建设"
